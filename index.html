The problem

The protocols are multiplying. The identity layer above them isn't.

Today, agents borrow financial identity from humans — shared API keys, shared cards, shared billing accounts. When something goes wrong, there's no per-agent credential to revoke, no atomic gate before the charge, and no shared reputation that follows the agent across rails.

01

Agents hold raw credentials

Developers embed real card numbers or raw API keys in agent context. A single prompt injection or context leak exposes the real account. There is no scoped credential an agent can hold safely.

02

Six protocols, six silos

ACP, AP2, TAP, Agent Pay, KYAPay, x402 each solve one execution layer. Each has its own agent registration and credential format. There is no unified identity layer above any of them.

03

No atomic pre-transaction gate

Networks authorise after the transaction is submitted. There is no single gate that enforces authority atomically before the upstream call — so concurrent agents race and overspend.

04

Inference and commerce are separate surfaces

An agent running Claude and completing a purchase operates under two unrelated caps with no shared enforcement. No unified view of agent economic activity across both surfaces.

05

Agent reputation is not portable

Mastercard, Visa, and Stripe each maintain their own agent registry. A behavioural profile built on one rail is invisible on another. Merchants cannot screen agents before fulfilling high-value orders.

"Every financial system built in the last 50 years assumes a human is the economic actor. AI agents are not humans. They need a financial identity of their own."

How it works

Three pieces. Two stay constant. The rail underneath changes.

RunVault sits between your agent and every execution mechanism. The identity and the atomic gate never change. The rail beneath them does — virtual card today, Stripe ACP, Visa VIC, AP2, x402 tomorrow.

01

Financial identity

A scoped JWT issued to the agent at initialisation — carrying agent_id, authority, scope, and TTL. The agent is a named economic actor, not a human borrowing credentials. Revocable in under a second. Portable across every rail and provider.

02

Atomic authority gate

A single Redis Lua script checks and debits authority in one atomic operation before any spend executes — whether the agent is calling Claude or completing a purchase. One gate. Both surfaces. Physically impossible to race.

03

Execution rail abstraction

Virtual card, Stripe ACP, Visa VIC, AP2, x402, stablecoin — RunVault routes to whatever mechanism fits the merchant. New rails arrive as adapters. The identity and the gate are identical on every rail.
